# Phase 1: Data Preparation and EDA - Context

**Gathered:** 2026-05-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 1 delivers the Colab notebook `notebooks/01_EDA_Preprocessing.ipynb` with code to load the Amazon labeled fake reviews CSV, inspect schema, run EDA, clean missing/duplicate rows, and save stratified train/validation/test split files. It must not train models, extract BERT embeddings, create behavioral features, or run PCA.

</domain>

<decisions>
## Implementation Decisions

### Runtime and execution
- **D-01:** The notebook remains Colab-first and keeps the local-execution guard.
- **D-02:** Planning and implementation may edit notebook files locally, but must not run notebook cells or process the dataset locally without owner approval.
- **D-03:** All executable Phase 1 logic must be inside `notebooks/01_EDA_Preprocessing.ipynb`; no pipeline `.py` modules.

### Data contract
- **D-04:** Original dataset stays at `data/final_labeled_fake_reviews.csv` and must not be overwritten.
- **D-05:** Cleaned outputs must go under `data/processed/`.
- **D-06:** Phase 2 depends on stable split outputs from Phase 1, so filenames must be predictable.

### Split policy
- **D-07:** Use stratified train/validation/test split with fixed `SEED = 42`.
- **D-08:** Default split ratio is 70/15/15 unless changed in notebook config.
- **D-09:** Save split metadata with row counts, class ratios, selected schema columns, cleaning counts, and seed.

### EDA scope
- **D-10:** Required EDA includes label distribution, missing values, duplicate rows, review length distribution, and class imbalance summary.
- **D-11:** Rating, reviewer, product, and timestamp analysis are included only if matching columns exist.

### the agent's Discretion
- Exact helper function names, plot styling, and whether cleaned outputs are CSV only or CSV plus parquet are implementation choices.
- Schema inference can use candidate column names plus explicit override variables.

</decisions>
