## Task 3: FGSM and PGD Robustness Evaluation

**Type:** implementation  
**Files:** `notebooks/06_Adversarial_XAI.ipynb`

Implement adversarial feature-space evaluation:

- Sample a seeded test subset for robustness.
- Compute train feature min/max per PCA component for clamping.
- Implement FGSM against the DL model:
  - Compute gradient of focal/cross-entropy loss wrt PCA input.
  - Perturb by `epsilon * sign(gradient)`.
  - Clamp to train feature bounds.
- Implement PGD against the DL model:
  - Initialize from clean features.
  - Use `PGD_STEPS`, alpha `epsilon / 4`.
  - Project into epsilon ball and clamp to train feature bounds.
- Evaluate for each condition:
  - `clean`
  - `fgsm_eps_*`
  - `pgd_eps_*`
- For each condition, compute metrics for:
  - `dl_pso`
  - `final_ensemble`
- Metrics must include:
  - accuracy
  - macro_f1
  - precision_fake
  - recall_fake
  - f1_fake
  - roc_auc
  - pr_auc
  - confusion matrix values
  - epsilon/method/subset seed/subset size
- Save:
  - `reports/tables/phase6_robustness_metrics.csv`
  - `reports/tables/phase6_adversarial_config.csv`
  - `reports/figures/phase6_robustness_metric_drop.png`

**Acceptance criteria:**

- Robustness table compares clean vs FGSM/PGD.
- Metric drops are explicit and not hidden.
- No model retraining happens in Phase 6.
