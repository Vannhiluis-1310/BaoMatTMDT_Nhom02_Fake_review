# Phase 10 Plan: Pipeline Redesign and Ensemble Model Zoo

## Goal

Update the existing notebook pipeline so the final project no longer depends on the weak PCA-first CNN-BiLSTM path. The revised pipeline must compare tabular and sequence models fairly, test many ensemble types, and select both balanced and precision-first winners with recall-aware metrics. Phase 5 must be split into one notebook per major model plus separate ensemble notebooks, with `05_Hybrid_Ensemble.ipynb` kept as the orchestrator/summary.

## Non-Negotiable Constraints

- Do not run notebook cells, training, tuning, EDA or dataset processing locally.
- Keep executable pipeline source inside the existing notebooks under `notebooks/`.
- Do not create standalone `.py` modules for the main ML pipeline.
- Do not modify `data/final_labeled_fake_reviews.csv`.
- RAM 12GB remains a hard design constraint for all proposed Colab code.

## Design Decisions

1. **ModernBERT location**
   - Phase 2 tabular track: ModernBERT tokenizer/model -> pooled 768-dim text embedding.
   - Phase 4 sequence track: ModernBERT tokenizer -> `input_ids` and `attention_mask` for CNN-BiLSTM sequence model.
   - Optional heavy variant: ModernBERT hidden states -> CNN-BiLSTM, only if Colab resources allow.

2. **Feature tracks**
   - Tabular: raw 777 = ModernBERT pooled 768 + 9 behavioral.
   - Sequence: token sequence + behavioral late fusion.
   - Diagnostic: PCA/SVD/Autoencoder for ablation only.

3. **Model candidates**
   - LightGBM raw 777.
   - XGBoost raw 777.
   - MLP raw 777.
   - CNN-BiLSTM-Attention sequence branch.
   - Optional PCA/Autoencoder variants for diagnostic tables.

4. **Ensemble candidates**
   - Single models.
   - Equal soft voting.
   - Weighted blending with grid weights summing to 1.
   - PSO weights only if grid is insufficient and runtime allows.
   - Stacking meta-models: Logistic/Ridge first, optional RF/LightGBM meta.
   - Calibration: sigmoid/Platt and isotonic where validation size is sufficient.

5. **Phase 5 notebook split**
   - Each major model gets a separate runnable Colab notebook.
   - Blending/stacking/calibration gets separate notebook(s).
   - The main `05_Hybrid_Ensemble.ipynb` reads all Phase 5 outputs and writes the final Phase 5 leaderboard.
